# Roadmap

## Near-Term

1. Stabilize the public `v0.6` draft package.
2. Improve examples so they demonstrate product-facing clarity, not only
   structural correctness.
3. Clarify the authoring profile for typed values, units, semantic IDs, and
   supporting context.
4. Clarify the boundary between the public spec, reference tooling, and
   deterministic implementation support after the first package has landed
   cleanly.

## Likely Next Spec Work

- tighten `pbc:config` guidance
- refine `pbc:grounding` and semantic anchor rules
- improve the line between canonical contract and supporting context
- strengthen examples for workflows, rules, and stable configuration
- add clearer public guidance for Markdown + Mermaid + progressive formalization

## Not The Goal Of This Repo

This roadmap does not include:

- hosted workflow/runtime features
- database or persistence design
- semantic job routing
- repo-ingestion product behavior
- implementation or runtime orchestration concerns

Those belong to the product and runtime layer, not the public spec repo.
