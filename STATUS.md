# Status

**Repo:** `pbc-spec`  
**Phase:** first public spec package  
**Spec version:** `0.6.0-draft`  
**Date:** March 2026

## Current State

This repo is ready to act as the first public home for:

- the current PBC draft specification
- the Markdown-first philosophy behind it
- a small set of worked examples
- a short roadmap for the next public iterations

## What Is Stable Enough To Review

- the core idea of PBC as the canonical authored `WHAT` layer
- the Markdown-first + structured-block authoring model
- the current `v0.6` direction around:
  - semantic anchors
  - supporting context blocks
  - `pbc:config`
  - lightweight `pbc:grounding`

## What Is Still Draft

- the exact long-term block set
- the final versioning/story around future `v1.0`
- the best authoring profile for typed values and richer semantic rigor
- the full relationship between spec, deterministic engine, and hosted runtime
- the CLI and viewer (reference implementations; not production-grade tooling)

## What Is Intentionally Missing Here

- the engine code
- deterministic parser/linter behavior
- hosted runtime or semantic orchestration
- product UX, persistence, or customer workflow

Those are intentionally kept outside this repo.
